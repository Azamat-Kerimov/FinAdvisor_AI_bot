# FinAdvisor — дизайн и UX

## 1. Анализ референсов (Monarch)

### Паттерны дизайна
- **Карточки по типам потоков:** Income (зелёный фон), Expenses (розовый), Savings (голубой) — быстрый скан без чтения подписей.
- **Чёткая иерархия:** Крупная сумма → подпись → контекст (период). Заголовок экрана + период сверху.
- **Сегментированные переключатели:** Net Worth / Cash, 1M / 3M / 6M — один ряд, без лишних кликов.
- **Прогресс онбординга:** «Getting Started» с чеклистом и прогресс-баром — понятно, что делать дальше.
- **Форма транзакции:** Лейбл слева, значение справа, разделители между полями. Дебит/Кредит — переключатель вверху.
- **Нижняя навигация:** 5 пунктов, иконка + текст, активное состояние явно выделено.
- **Минимум декора:** Белый/серый фон, пастельные акценты только для смысла (доход/расход/сбережения).

### Что можно улучшить
- **Десктоп:** Референсы заточены под мобильный вид; нужен адаптив: сайдбар вместо bottom nav, многоколоночные карточки.
- **Читаемость аналитики:** График без подписей осей и тултипов — добавить подписи и hover.
- **Единый язык карточек:** Разные отступы и тени у списков — унифицировать один стиль карточки (радиус, тень, отступ).
- **Контраст и доступность:** Проверить соотношение текста и фона по WCAG.
- **Масштабирование:** Виджеты (Getting Started, Calendar) — вынести в переиспользуемые блоки с пропсами.

## 2. Структура страниц (Layout)

```
┌─────────────────────────────────────────┐
│  Header (логотип / заголовок экрана)   │
├─────────────────────────────────────────┤
│                                         │
│  Контент экрана (scroll)                │
│  - Dashboard: Cash flow карточки,       │
│    краткий инсайт, быстрые действия      │
│  - Транзакции: фильтры, сводка, список │
│  - Бюджеты: статус по категориям        │
│  - Капитал: активы / долги              │
│  - Консультация: чат с ИИ               │
│  - О приложении: текст + удаление данных│
│                                         │
├─────────────────────────────────────────┤
│  Bottom Nav (mobile) / Sidebar (desktop)│
└─────────────────────────────────────────┘
```

- **Главный экран (Dashboard):** период (текущий месяц), три карточки: Доходы / Расходы / Остаток (или «Сбережения»). Ниже — короткий инсайт (топ расходов) и рекомендация по резерву. Без лишних виджетов на первом экране.
- **Адаптив:** до ~768px — bottom nav; от 768px — боковая панель, контент в max-width контейнере.

## 3. Масштабирование дизайна

- **Новые блоки на Dashboard:** Добавлять как `<section>` с общим классом карточки (например `StatCard`) и едиными отступами.
- **Новые экраны:** Подключать в роутер и в `BottomNav`/`Sidebar`; контент оборачивать в `AppLayout`.
- **Темы:** Цвета вынесены в Tailwind theme (см. `tailwind.config.js`); тёмная тема — через `dark:` и один переключатель.
- **Компоненты UI:** Кнопки, инпуты, карточки — в `components/ui`; переиспользовать, не дублировать разметку.
