# Рекомендации по продукту FinAdvisor

## 1. Интерфейс и мобильная версия (UI/UX и телефон)

Большая часть пользователей будет заходить с телефона. Ниже — что уже хорошо и что стоит доработать.

### Что уже хорошо

- **Нижняя навигация** — фиксированный бар внизу с иконками и подписями, учёт `safe-area-inset-bottom` для выреза экрана.
- **Ограничение ширины контента** — `max-w-[480px]` по центру, читаемо и на телефоне, и на планшете.
- **Таблицы** — у блоков «Ваш прогресс», «Сравнение с целевыми нормами», «Арена/покупка», «Влияние факторов» есть `overflow-x-auto`, горизонтальный скролл на узком экране работает.
- **Модальные окна** — затемнение фона, `max-h-[90vh]`, скролл внутри карточки (импорт транзакций, формы добавления).
- **Кнопки действий** — основные CTA полноширинные (`w-full`) или с достаточным отступом, удобно нажимать пальцем.
- **Помощь** — раздел с раскрывающимися блоками, один пункт = одна тема.

### Мобильные недостатки и правки

1. **Viewport: `user-scalable=no`** — *выполнено*: в `index.html` используется `width=device-width, initial-scale=1.0` без `user-scalable=no`, зум разрешён.

2. **Нижняя навигация: 7 пунктов на узком экране**  
   У каждого пункта `min-w-[72px]`, всего 7×72 ≈ 504px. На ширине 320–360px контент навигации не помещается, возможен перенос или сжатие.  
   **Рекомендации (на выбор):**  
   - уменьшить `min-w` на малых экранах (например, до 44–48px) и/или уменьшить размер иконок/шрифта;  
   - либо сворачивать часть пунктов в «Ещё» с выпадающим меню;  
   - либо делать горизонтальный скролл бара (менее удобно для главной навигации).

3. **Верхняя безопасная зона (вырез/динамический остров)**  
   Сейчас у контента `pt-[3.25rem]` без учёта `env(safe-area-inset-top)`. На аппаратах с вырезом контент может заходить под статус-бар.  
   **Рекомендация:** добавить отступ сверху с учётом safe area, например:  
   `pt-[max(3.25rem,calc(3.25rem+env(safe-area-inset-top)))]` или вынести в переменную в `AppLayout`.

4. **Размер зон нажатия (touch targets)**  
   Рекомендуемый минимум ~44×44px. Сейчас:  
   - кнопки «?» (подсказки) — `w-6 h-6` (24px);  
   - чекбоксы фильтров (период, категория, «Исключить переводы») — `h-4 w-4` (16px);  
   - иконки «Редактировать»/«Удалить» в списке транзакций — кнопка `p-2.5`, область приемлемая, но можно чуть увеличить.  
   **Рекомендации:**  
   - для кнопок «?» добавить внутренний отступ (padding), чтобы зона нажатия была не меньше 44px при сохранении визуального размера 24px;  
   - у чекбоксов обернуть в `<label>` с `py-2` / `min-h-[44px]` и растягивать кликабельную область на всю строку (у вас часть уже с label — проверить все фильтры).

5. **Высота полей ввода на мобильных**  
   Часть полей с `py-2` или `py-1.5` даёт высоту меньше 44px.  
   **Рекомендация:** для основных полей ввода (поиск, сумма, категория, формы в модалках) задать `min-h-[44px]` или `py-3`, чтобы удобнее попадать пальцем.

6. **Мелкий шрифт в таблице «Влияние факторов»**  
   В сценариях используется `text-[10px]`. На телефоне тяжело читать.  
   **Рекомендация:** на экранах до 400px поднять до `text-xs` (12px), таблицу оставить с `overflow-x-auto`.

7. **Закрытие модалок**  
   Закрытие по клику на затемнение реализовано. Дополнительно можно закрывать по свайпу вниз или по кнопке «Назад» в браузере (History API), чтобы поведение было привычным на мобильных.

8. **Кнопка закрытия модалки**  
   Кнопки «✕» в модалках без явного размера; лучше задать `min-w-[44px] min-h-[44px]` и центрировать иконку, чтобы их было проще нажимать.

### Общие улучшения интерфейса

- **Состояния загрузки:** где есть асинхронные действия (консультация, импорт, сохранение) — уже есть спиннеры/блокировки; при желании можно добавить скелетоны вместо пустых блоков.
- **Пустые состояния:** на главной и в разделах при отсутствии данных уже есть тексты вроде «Нет данных»; можно добавить короткий призыв к действию (например, «Добавьте первую транзакцию» со ссылкой на раздел).
- **Ошибки сети:** при падении API показывать понятное сообщение и кнопку «Повторить», особенно важно на мобильном при нестабильном интернете.
- **Выбор месяца на главной:** селект месяца можно заменить на стрелки «← месяц / месяц →» или горизонтальный скролл месяцев — на телефоне так часто удобнее, чем нативный `<select>`.
- **Сценарии:** заголовок страницы «Сценарный анализ» совпадает с названием второго блока; можно переименовать страницу, например, в «Калькуляторы и сценарии», чтобы отличать от блока «Сценарный анализ».

### Идеи фич (по приоритету)

- **Pull-to-refresh** на главной и в списках (транзакции, капитал) — привычный жест на мобильных для обновления данных.
- **Быстрые действия** — с главной переход не только по шагам онбординга, но и по одному тапу: «Добавить расход», «Загрузить выписку», «Получить консультацию».
- **Экспорт/шаринг** — выгрузка отчёта (PDF/Excel) или шаринг картинки «Мой прогресс» в соцсети/мессенджеры.
- **Виджеты/напоминания** — «Не забыли внести расходы за неделю?» или «Резерв ниже 3 месяцев — пополните подушку».
- **Тёмная тема** — переключатель в профиле или по системной настройке (`prefers-color-scheme`).

**Итог по мобильной версии:** разрешить масштабирование, поправить нижний бар на узких экранах, добавить safe area сверху, увеличить зоны нажатия у «?» и чекбоксов, поднять минимальную высоту полей ввода и размер шрифта в таблице факторов. Остальная база (навигация, таблицы, модалки, помощь) уже ориентирована на мобильное использование.

## 3. Интеграция с банками (Сбер и Т-Банк)

### Варианты интеграции

1. **Официальные API банков (если доступны)**
   - **Сбер:** СберБанк API для бизнеса — доступ к выпискам и операциям по счёту. Требует заключения договора, подключения как юрлицо/ИП и прохождения модерации. Подходит для B2B, для физлиц обычно недоступно напрямую.
   - **Т-Банк:** Тинькофф Инвестиции API и Open API — в основном для брокерских счётов и карт. Для выписок по картам/счёту нужно уточнять наличие открытого API для физлиц.

2. **Агрегаторы и партнёрские решения**
   - **ЦБ РФ / Финтех:** сервисы агрегации финансовых данных (например, на базе СБП или согласий пользователя). Требуют статуса участника экосистемы и соблюдения регуляторики.
   - **Партнёрство с банком:** договор с банком на доступ к данным клиента по его согласию (OAuth/consent). Даёт легальный и стабильный доступ, но требует юридической и технической проработки.

3. **Практичный путь без прямого API**
   - **Импорт выписок (уже есть):** пользователь выгружает Excel из приложения банка (Сбер, Т-Банк) и загружает в FinAdvisor. Минус — ручной шаг; плюс — не нужны договоры с банками и не нужен доступ к API.
   - **Улучшение импорта:** поддержка дополнительных форматов (CSV, QIF), распознавание колонок по заголовкам, запоминание маппинга категорий для конкретного банка. Это повышает ценность без интеграции по API.
   - **Ссылка на инструкции:** явная ссылка на видео/статью «Как выгрузить выписку из Сбера / Т-Банка» в интерфейсе загрузки файла.

4. **Будущее: Open Banking**
   - В РФ развивается идея открытых банковских API (по аналогии с PSD2 в ЕС). Следить за инициативами ЦБ и банков — при появлении единого механизма доступа по согласию пользователя можно будет подключить один контур и получать данные из нескольких банков.

**Итог:** пока нет договора с банком или агрегатором, оптимально развивать импорт выписок (форматы, маппинг, онбординг) и готовить архитектуру к будущему подключению API (отдельный модуль «банковские подключения», OAuth-поток, хранение токенов по стандартам безопасности).

## 5. Бесплатный период и монетизация

### Текущая модель
- 2 бесплатных месяца для новых пользователей.
- Далее подписка 299 ₽/мес или 2990 ₽/год.

### Рекомендации

1. **Оставить 2 месяца бесплатно — разумно**
   - За 2 месяца пользователь успевает: заполнить данные, получить 2–4 консультации ИИ, увидеть ценность (графики, цели, рекомендации). Конверсия в платящих выше, чем при 7–14 днях.
   - Уменьшать до 1 месяца можно для A/B-теста, если конверсия в оплату низкая и есть гипотеза «короткий триал — меньше откладываний».
   - Увеличивать до 3 месяцев имеет смысл только при сильном акценте на виральность («пригласи друга — получи ещё месяц») или при очень дорогом продукте.

2. **Платные только консультации — альтернатива**
   - Сделать бесплатными: учёт транзакций, капитал, цели, дашборд, бенчмарки, алерты.
   - Платить только за консультации ИИ: например, 5 консультаций в месяц входят в подписку 199 ₽/мес, либо пакет «10 консультаций за 299 ₽» без подписки.
   - Плюсы: низкий порог входа, пользователь привыкает к продукту на бесплатных фичах и платит за «ум» (ИИ). Минусы: нужен чёткий лимит бесплатных консультаций (0 или 1 в месяц), иначе расходы на GigaChat съедят выручку.

3. **Гибрид**
   - Бесплатно: 1 консультация в месяц, остальной функционал без ограничений.
   - Подписка: 5 консультаций в месяц + приоритетная генерация, экспорт отчётов, «цель месяца» и чек-листы из консультации.
   - Так сохраняется ценность бесплатного периода и понятное усиление при оплате.

**Итог:** оставить 2 бесплатных месяца и текущие тарифы — обоснованно. При желании усилить конверсию можно A/B тестировать: 1 месяц бесплатно vs 2; либо ввести тариф «только консультации» (пакет запросов к ИИ без подписки на весь продукт).
