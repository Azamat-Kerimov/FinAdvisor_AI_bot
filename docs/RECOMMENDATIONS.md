# Рекомендации по продукту FinAdvisor

## 3. Интеграция с банками (Сбер и Т-Банк)

### Варианты интеграции

1. **Официальные API банков (если доступны)**
   - **Сбер:** СберБанк API для бизнеса — доступ к выпискам и операциям по счёту. Требует заключения договора, подключения как юрлицо/ИП и прохождения модерации. Подходит для B2B, для физлиц обычно недоступно напрямую.
   - **Т-Банк:** Тинькофф Инвестиции API и Open API — в основном для брокерских счётов и карт. Для выписок по картам/счёту нужно уточнять наличие открытого API для физлиц.

2. **Агрегаторы и партнёрские решения**
   - **ЦБ РФ / Финтех:** сервисы агрегации финансовых данных (например, на базе СБП или согласий пользователя). Требуют статуса участника экосистемы и соблюдения регуляторики.
   - **Партнёрство с банком:** договор с банком на доступ к данным клиента по его согласию (OAuth/consent). Даёт легальный и стабильный доступ, но требует юридической и технической проработки.

3. **Практичный путь без прямого API**
   - **Импорт выписок (уже есть):** пользователь выгружает Excel из приложения банка (Сбер, Т-Банк) и загружает в FinAdvisor. Минус — ручной шаг; плюс — не нужны договоры с банками и не нужен доступ к API.
   - **Улучшение импорта:** поддержка дополнительных форматов (CSV, QIF), распознавание колонок по заголовкам, запоминание маппинга категорий для конкретного банка. Это повышает ценность без интеграции по API.
   - **Ссылка на инструкции:** явная ссылка на видео/статью «Как выгрузить выписку из Сбера / Т-Банка» в интерфейсе загрузки файла.

4. **Будущее: Open Banking**
   - В РФ развивается идея открытых банковских API (по аналогии с PSD2 в ЕС). Следить за инициативами ЦБ и банков — при появлении единого механизма доступа по согласию пользователя можно будет подключить один контур и получать данные из нескольких банков.

**Итог:** пока нет договора с банком или агрегатором, оптимально развивать импорт выписок (форматы, маппинг, онбординг) и готовить архитектуру к будущему подключению API (отдельный модуль «банковские подключения», OAuth-поток, хранение токенов по стандартам безопасности).

## 5. Бесплатный период и монетизация

### Текущая модель
- 2 бесплатных месяца для новых пользователей.
- Далее подписка 299 ₽/мес или 2990 ₽/год.

### Рекомендации

1. **Оставить 2 месяца бесплатно — разумно**
   - За 2 месяца пользователь успевает: заполнить данные, получить 2–4 консультации ИИ, увидеть ценность (графики, цели, рекомендации). Конверсия в платящих выше, чем при 7–14 днях.
   - Уменьшать до 1 месяца можно для A/B-теста, если конверсия в оплату низкая и есть гипотеза «короткий триал — меньше откладываний».
   - Увеличивать до 3 месяцев имеет смысл только при сильном акценте на виральность («пригласи друга — получи ещё месяц») или при очень дорогом продукте.

2. **Платные только консультации — альтернатива**
   - Сделать бесплатными: учёт транзакций, капитал, цели, дашборд, бенчмарки, алерты.
   - Платить только за консультации ИИ: например, 5 консультаций в месяц входят в подписку 199 ₽/мес, либо пакет «10 консультаций за 299 ₽» без подписки.
   - Плюсы: низкий порог входа, пользователь привыкает к продукту на бесплатных фичах и платит за «ум» (ИИ). Минусы: нужен чёткий лимит бесплатных консультаций (0 или 1 в месяц), иначе расходы на GigaChat съедят выручку.

3. **Гибрид**
   - Бесплатно: 1 консультация в месяц, остальной функционал без ограничений.
   - Подписка: 5 консультаций в месяц + приоритетная генерация, экспорт отчётов, «цель месяца» и чек-листы из консультации.
   - Так сохраняется ценность бесплатного периода и понятное усиление при оплате.

**Итог:** оставить 2 бесплатных месяца и текущие тарифы — обоснованно. При желании усилить конверсию можно A/B тестировать: 1 месяц бесплатно vs 2; либо ввести тариф «только консультации» (пакет запросов к ИИ без подписки на весь продукт).
